# 14. 멀티 모달 LLM

## 14.1 멀티 모달 LLM이란

**멀티 모달 LLM**: 다양한 형식의 데이터를 이해하고 생성할 수 있는 LLM

### 14.1.1 멀티 모달 LLM의 구성요소

![alt text](<../static/img 14.2 멀티 모달 LLM의 구성요소.png>)
> 멀티 모달 LLM의 구성 요소

- 모달리티 인코더(modality encoder), 입력 프로젝터(input projector)
    - 이미지 형식의 데이터를 텍스트로 변환
- 출력 프로젝터(output projector) 
    - 이미지 형태의 데이터 출력이 필요한지 판단
- 모달리티 생성기(modality generator)
    - 특정 데이터 형식의 출력을 생성
> 입력 프로젝터와 출력 프로젝터에서는 파라미터를 업데이트함으로써 멀티 모달 이해와 생성 성능을 높인다.

<br>

**모달리티 인코더**: 텍스트 이외의 데이터 형식을 처리하기 위해 학습된 사전 학습 모델
- 비전 트랜스포머(vision transformer): 텍스트를 처리하기 위해 개발된 트랜스포머 아키텍처를 이미지에 적용한 모델
![alt text](<../static/img 14.3 비전 트랜스포머 작동 방식.png>)
    > 비전 트랜스포머 작동 방식
- CLIP(Contrastive Language-Image Pre-training) 모델: 인터넷상에서 수집한 이미지와 캡션 데이터를 활용해 이미지와 텍스트를 같은 벡터 공간에 임베딩하도록 만들어진 모델
    - 내부적으로 이미지 처리 모델과 텍스트 처리 모델이 함께 사용

**입력 프로젝터**: 이미지 임베딩을 LLM 백본이 이해할 수 있는 텍스트로 변환

**출력 프로젝터**: 이미지 생성이 필요한지 판단하고 어떤 이미지를 생성할지 정한다.

**모달리티 생성기**: 출력 프로젝터를 통해 생성된 이미지 생성 프롬프트를 전달 받아 이미지 생성
